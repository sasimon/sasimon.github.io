The Elements of Form Design

The main idea in designing forms is to make them easy to use for the users. I considered that a form should be easy to read by separating the content into discrete blocks. To me, this was better than the jumble of text and fields that I saw when I first loaded Millennium Computers' Web page into my Web browser.

As for the elements themselves, I feel that the tabular layout imitated by floating the elements left and right does justice. I mean the ayout that floats the labels left and the fields right. Most people consider a "tabular" layout to refer to a datasheet that can display multiple records, but a Web site form cannot do that. It is mainly to submit a single record.

The only other thing I feel the need to say is that I elected not to have any in-line validation in my code. It is not that it is not part of the CSS3 standard. It is in fact part of the standard, but the W3C CSS Validator does not recognize the :required, :valid, and :invalid pseudo-classes. Next semester I will learn JavaScript as it relates to Web design, and it will help me imitate the valid/invalid pseudo-class events. But for now, I just have to do without it.
